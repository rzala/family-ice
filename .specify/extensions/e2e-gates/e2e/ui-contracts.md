# UI Test Contracts

Required `data-testid` (or configured convention) attributes that frontend components **must** include for E2E gate tests to work.

Generated by `/speckit.e2e.init`. Update this file when adding new gate tests.

## Auth / Login Screen

| Attribute | Element | Used By |
|-----------|---------|---------|
| `login-page` | Login page container | gate-ui-readability |
| `login-email-input` | Email input field | gate-ui-readability |
| `login-submit` | Submit / magic link button | gate-ui-readability |

## Protected / Dashboard Screen

| Attribute | Element | Used By |
|-----------|---------|---------|
| `dashboard-page` | Dashboard page container | — |
| `dashboard-nav` | Dashboard navigation | — |
| `dashboard-heading` | Main heading (h1/h2) | gate-ui-readability |
| `dashboard-anchor` | Key anchor element on protected route | gate-auth, gate-ui-readability |

## Common Layout

| Attribute | Element | Used By |
|-----------|---------|---------|
| `app-root` | Application root container | — |
| `main-content` | Main content area | gate-ui-readability |
| `nav-bar` | Navigation bar | gate-ui-readability |
| `error-boundary` | Error boundary fallback | — |

## Adding New Contracts

1. Add the selector to `selectors.ts`
2. Add a row to this table
3. Add the `data-testid` attribute to the frontend component
4. Add assertions in the appropriate gate test
